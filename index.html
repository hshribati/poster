Soteria

Smart Emergency Monitoring & Evacuation System: Real-time PoE-powered hazard detection and dynamic evacuation guidance.

Problem Statement

Modern building emergency systems often fail to provide reliable real-time communication. Existing systems operate in isolated components, causing delays in hazard signal delivery that impact occupant decision-making[cite: 9, 10, 11].

Project History

Launched in 2019 as an IPRO 497 initiative, Soteria has grown through multidisciplinary teams. Spring 2026 introduces a full networked system with GraphQL, Raspberry Pi controllers, and real-time dashboards[cite: 17, 19, 20].

How It Works

1
Hazard Detected: Smoke/CO sensors send data via Ethernet to the Raspberry Pi[cite: 23, 24].
2
Risk Analysis: Model server classifies unsafe zones and applies evacuation logic[cite: 25, 26].
3
LED Guidance: Red indicates danger; Green shows safe paths—visible in low visibility[cite: 27, 28].
4
Dashboard Update: Clients receive real-time updates via WebSocket subscriptions[cite: 29, 30].

Limitations

  • Documentation gaps from previous teams created initial knowledge hurdles[cite: 75].
  • Legacy hardware limited available computing power and sensor compatibility[cite: 79].
  • Budget constraints and parts delays impacted testing and iteration cycles[cite: 80, 81].

System Design Overview

The system utilizes a PoE switch to deliver 48V over standard Ethernet cabling, powering all nodes via splitters. The central GraphQL server manages state and broadcasts evacuation directives[cite: 34, 36, 59].

Software Architecture

  • GraphQL API: Manages queries and real-time WebSocket subscriptions[cite: 61].
  • React Dashboard: Visualizes sensor readings and LED states in real time[cite: 62, 63].
  • Deployment: Containerized via mkasi tool for direct hardware deployment[cite: 64].
