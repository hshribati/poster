Soteria [cite: 1]

Smart Emergency Monitoring & Evacuation System [cite: 2]

Problem Statement [cite: 8]

Modern building emergency systems often fail to provide reliable real-time communication during critical situations[cite: 9]. Existing systems operate in isolated components, causing delays and inconsistent device updates across connected endpoints[cite: 10].

Objective [cite: 12]

Design and improve a PoE-powered networked emergency monitoring system that detects hazardous conditions—smoke, CO, and elevated temperature [cite: 13]—and dynamically guides occupants to safety via LED evacuation paths[cite: 14].

Project History [cite: 16]

Launched in 2019 as an IPRO 497 initiative[cite: 17]. Spring 2026 introduced a full networked system with GraphQL, Raspberry Pi controllers, and real-time dashboards[cite: 19, 20].

How It Works [cite: 21]

1
Hazard Detected: Smoke/CO sensors send data via Ethernet to the local Raspberry Pi controller[cite: 24].
2
Risk Analysis: Model server classifies unsafe zones from live readings and applies evacuation logic[cite: 26].
3
LED Guidance: RED danger zones and GREEN safe evacuation paths are visible in low-visibility conditions[cite: 28].
4
Dashboard Update: Clients receive status updates via WebSocket subscriptions within milliseconds[cite: 30].

System Design Overview [cite: 31]

A PoE switch delivers 48V over standard Ethernet cabling, powering all nodes via splitters[cite: 34]. The system integrates distributed hardware sensors and embedded Raspberry Pi controllers with a centralized GraphQL model server[cite: 33].

Software Architecture [cite: 47]

  • Centralized Model Server: GraphQL-based service managing real-time state for all devices[cite: 59].
  • GraphQL API: Supports queries, mutations, and live updates via WebSocket subscriptions[cite: 61].
  • React Dashboard: Displays sensor readings, danger flags, and LED states in real time[cite: 62].
  • Deployment: Containerized via mkasi tool, deployable directly to Raspberry Pi hardware[cite: 64].
