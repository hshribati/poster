Soteria

Smart Emergency Monitoring & Evacuation System

Problem Statement

Modern building emergency systems often fail to provide reliable real-time communication during critical situations. Existing systems operate in isolated components, causing delays and inconsistent device updates across connected endpoints.

Objective

Design and improve a PoE-powered networked emergency monitoring system that detects hazardous conditions—smoke, CO, and elevated temperature—and dynamically guides occupants to safety via LED evacuation paths.

Project History

Launched in 2019 as an IPRO 497 initiative. Spring 2026 introduced a full networked system with GraphQL, Raspberry Pi controllers, and real-time dashboards.

How It Works

1
Hazard Detected: Smoke/CO sensors send data via Ethernet to the local Raspberry Pi controller.
2
Risk Analysis: Model server classifies unsafe zones from live readings and applies evacuation logic.
3
LED Guidance: RED danger zones and GREEN safe evacuation paths are visible in low-visibility conditions.
4
Dashboard Update: Clients receive status updates via WebSocket subscriptions within milliseconds.

System Design Overview

A PoE switch delivers 48V over standard Ethernet cabling, powering all nodes via splitters. The system integrates distributed hardware sensors and embedded Raspberry Pi controllers with a centralized GraphQL model server.

Software Architecture

  • Centralized Model Server: GraphQL-based service managing real-time state for all devices.
  • GraphQL API: Supports queries, mutations, and live updates via WebSocket subscriptions.
  • React Dashboard: Displays sensor readings, danger flags, and LED states in real time.
  • Deployment: Containerized via mkasi tool, deployable directly to Raspberry Pi hardware.

Limitations

  • Legacy Raspberry Pi models limited available computing power.
  • Significant time spent diagnosing inherited models lacking sensors.
  • Budget constraints prevented experimentation to find optimal components.
