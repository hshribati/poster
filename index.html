Soteria

Smart Emergency Monitoring & Evacuation System: Real-time PoE-powered hazard detection and dynamic evacuation guidance for safety-critical buildings[cite: 6].

Problem Statement

Modern building emergency systems often fail to provide reliable real-time communication during critical situations[cite: 9]. Isolated components cause delays and inconsistent device updates across connected endpoints[cite: 10].

Objective

Design a PoE-powered networked system that detects smoke, CO, and elevated temperatures[cite: 13, 14]. This semester focuses on resolving real-time synchronization between distributed hardware, GraphQL, and dashboard clients[cite: 15].

How It Works

1
Hazard Detected: Sensors send data via Ethernet to the local Raspberry Pi[cite: 24].
2
Risk Analysis: Model server classifies unsafe zones and applies evacuation logic[cite: 26].
3
LED Guidance: Red for danger, Green for safe paths—visible in low visibility[cite: 28].
4
Dashboard Update: Real-time status updates via WebSocket subscriptions[cite: 30].

System Design Overview

Architecture Diagram [cite: 32]

A PoE switch delivers 48V over standard Ethernet, powering all nodes via splitters. Each Pi monitors its zone and reports to the central server[cite: 35].
